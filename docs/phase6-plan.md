# NexusMind Phase 6 实施计划（稳定性）

## 1. 目标范围

- 建立 Chrome/Edge + 典型 SPA 固定回归集并固化报告模板
- 增强流式断流监控与重连策略，补齐异常链路观测
- 完成计费并发压测与风控分层（降级/白名单/人工复核入口）
- 建立稳定性门禁，形成发布前看板化检查

## 2. 里程碑

- M1：流式问答链路支持断流重试与重连指标采集
- M2：计费状态机支持 requestId 幂等计量与分层风控
- M3：侧边栏接入稳定性看板、风控复核入口
- M4：输出固定回归集与 Phase 6 稳定性报告，更新发布门禁

## 3. 风险清单

- 风险：OpenAI SSE 不支持断点续传，重连后可能出现内容重算
  - 缓解：限制重连次数，重连态显式提示，失败后保留可诊断错误摘要
- 风险：浏览器 service worker 生命周期导致重试状态丢失
  - 缓解：关键指标持久化到 `chrome.storage.local`
- 风险：本地风控阈值可能对特定高频合法场景产生误伤
  - 缓解：增加白名单通道与人工复核工单入口
- 风险：跨浏览器自动化回归尚未接入 CI
  - 缓解：先执行固定手工回归集并文档化，Phase 7 再自动化

## 4. 验收口径（DoD）

- 功能完成：
  - 流式断流自动重连可用，UI 可见重连状态
  - 稳定性看板可查看流式与风控核心指标
  - 计量幂等生效，重复 requestId 不重复扣费
  - 风控支持 none/degraded/blocked 分层、白名单与人工复核入口
- 测试通过：
  - 单元/集成测试覆盖新增计费幂等与风控分层逻辑
  - 全量 `npm test`、`npm run typecheck`、`npm run build` 通过
- 性能达标：
  - 继续满足页面重写 ≤1s（不含 AI 调用）
  - 图谱 2000 节点性能基线通过
- 文档完成：
  - README、architecture、billing、security、assumptions、changelog 同步更新
  - 新增 Phase 6 计划与回归报告文档
- 回滚明确：
  - 可关闭重连策略并回退到单次流式请求
  - 风控策略可回退为单阈值阻断模式
