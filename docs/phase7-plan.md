# NexusMind Phase 7 实施计划（发布能力）

## 1. 目标与范围

- 建立 Chrome/Edge 自动化回归矩阵，并接入 CI 门禁。
- 补齐发布文档与用户手册，落地上架前检查自动化。
- 将订阅校验从本地规则迁移为服务端签名校验。
- 提供开发人员手册，规范后续维护与开发流程。

## 2. 里程碑

### M1 自动化基线

- 新增 CI 工作流，覆盖 typecheck/test/build 基础门禁。
- 新增扩展级自动化回归流程，矩阵执行 Chromium 与 Edge。
- 将回归结果产物化，支持发布前追溯。

### M2 发布文档完善

- 新增用户手册（安装、核心流程、排障）。
- 新增开发手册（架构边界、开发流程、测试规范）。
- 升级发布检查清单，接入自动化检查脚本。

### M3 订阅校验升级

- 设计并落地签名负载 Schema 与校验器。
- 后台订阅激活流程改为签名验证路径。
- 增补单元测试覆盖有效签名、过期签名、篡改签名、白名单声明。

### M4 收口与验收

- 执行 typecheck/test/build 与浏览器回归。
- 输出测试/性能/安全/兼容性结论与未决风险。
- 更新 README、changelog、assumptions 文档形成闭环。

## 3. 风险清单与缓解策略

- 风险：CI 中 Edge 通道与扩展加载存在平台差异。
  - 缓解：采用 Chromium 与 Edge 分 job 运行，Edge 在 Windows runner 执行。
- 风险：签名校验引入后与旧 Token 不兼容。
  - 缓解：引入严格 Schema 与可观测错误信息，保留可回退配置记录到 assumptions。
- 风险：扩展自动化回归对页面时序敏感，偶发波动导致误报。
  - 缓解：使用本地可控 SPA 测试页面与稳定等待策略，降低外部依赖。

## 4. DoD（阶段验收口径）

1. 功能完成
   - CI 门禁上线且 PR/主分支均触发。
   - 发布文档、用户手册、开发手册齐备并可执行。
   - 订阅校验完成服务端签名路径迁移。
2. 测试通过
   - 单元、集成、E2E、构建测试全部通过。
3. 性能指标达标
   - 页面重写 1 秒、图谱 2000 节点、流式链路既有基准保持通过。
4. 文档更新完成
   - README、architecture、billing、release-checklist、assumptions、changelog 同步更新。
5. 可回滚方案明确
   - CI、签名校验、发布流程均有回退策略并记录于 assumptions 与清单。
