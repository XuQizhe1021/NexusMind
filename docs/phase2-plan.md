# Phase 2 实施计划（图谱基础）

## 1. 实施计划

1. 落地 `@nexusmind/graph` 的 IndexedDB（Dexie）数据层，建立 `Entity / Relation / Page` 三类核心模型与索引。
2. 建立“当前页文本 -> 实体关系抽取 -> 跨页归一 -> 图谱写入”的流水线，并在扩展后台通过消息接口开放能力。
3. 在侧边栏新增图谱面板，实现“收录当前页、关键词检索、关系回溯、最小可视化”。
4. 补齐图谱单元测试、集成测试与 2000 节点性能基准，并完成文档与阶段记录更新。

## 2. 里程碑

- M1：图谱数据模型与仓储完成，支持基础读写与按键查询。
- M2：跨页归一流水线完成，重复收录页面可正确覆盖旧索引。
- M3：侧边栏图谱能力完成，用户可执行收录与检索回溯。
- M4：测试与性能基准通过，文档齐全并可复现实验结果。

## 3. 风险清单

- 抽取质量风险：启发式实体抽取精度有限，可能引入噪声实体。
- 性能风险：大文本页面收录可能拖慢侧边栏交互响应。
- 兼容性风险：不同站点文本结构差异可能影响抽取稳定性。
- 数据一致性风险：页面重复收录时若增量回收不完整会导致“幽灵关系”。

## 4. 验收口径（DoD）

- 功能完成：具备实体/关系/Page 持久化、跨页归一、图谱搜索与回溯。
- 测试通过：图谱单元/集成/性能测试全部通过。
- 性能达标：2000 节点检索与回溯可在 1 秒内完成。
- 文档更新：README、architecture、assumptions、changelog 同步更新。
- 可回滚明确：单页重收录可覆盖旧索引，保留清空图谱接口以回退状态。
