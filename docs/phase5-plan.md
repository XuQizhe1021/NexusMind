# Phase 5 实施计划（商业化）

## 1. 实施计划

1. 在 `@nexusmind/billing` 落地订阅状态机，明确免费/订阅双轨与状态迁移规则。
2. 在 background 问答入口接入权限门控与调用计量，确保“会话完成才扣减”。
3. 在 sidepanel 增加商业化面板，提供订阅校验、增量包购买、取消与退款操作。
4. 补齐审计日志与风控策略，禁止敏感明文落库并输出最小必要信息。
5. 完成测试与文档同步，确保 DoD 可验收且具备可回滚路径。

## 2. 里程碑

- M1：billing 状态机、计量、增量包与退款取消流程可用。
- M2：问答链路门控生效，超额场景可返回购买引导。
- M3：商业化侧边栏操作闭环打通，状态可视化与审计日志可查看。
- M4：类型检查、测试、构建通过，文档与发布清单更新完成。

## 3. 风险清单

- 订阅校验来源风险：账户系统未落地前需采用可回退的本地校验策略。
- 计量一致性风险：并发调用下需避免重复扣减与漏扣减。
- 风控误伤风险：高频规则过严可能阻断正常用户。
- 退款争议风险：退款后权益回收时点与用户预期可能不一致。

## 4. 验收口径（DoD）

- 功能完成：订阅校验、权限门控、500 次月额度、增量包、退款取消、风控审计全部可用。
- 测试通过：计费核心逻辑单元测试通过，且全量测试与构建通过。
- 性能达标：页面重写 1 秒目标不回退，图谱 2000 节点能力不回退，流式问答仍可用。
- 文档更新：README、architecture、security、billing、assumptions、changelog、release-checklist 全部更新。
- 可回滚明确：订阅可切回免费轨道，取消可按周期生效，退款可回收权益并保留审计链路。
