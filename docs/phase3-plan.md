# Phase 3 实施计划（页面重写）

## 1. 实施计划

1. 扩展统一设置模型，新增页面重写多意图与站点默认意图配置，形成可校验、可持久化链路。
2. 在 content script 落地 DOM 重排引擎，支持学习/摘要/去干扰三类重写策略与一键还原。
3. 增加 SPA 路由监听与重写重入控制，确保路由切换自动回滚、同意图重复应用幂等。
4. 补齐页面重写规则测试、回滚稳定性测试与性能基准（不含 AI 调用）。

## 2. 里程碑

- M1：设置链路支持 `rewrite.defaultIntent + rewrite.siteIntents` 并完成 UI 配置入口。
- M2：DOM 重写引擎支持应用、回滚、状态查询，且可逆变更记录完整。
- M3：SPA 路由切换自动回滚，重写并发与重复触发可控。
- M4：测试矩阵通过，页面重写性能基准达到 1 秒内。

## 3. 风险清单

- 站点结构风险：通用内容区识别可能在复杂门户站点出现误判。
- SPA 事件风险：个别前端框架路由机制可能绕过标准 history 事件。
- 可逆性风险：第三方脚本在重写期间改写 DOM 时，回滚结果可能与初始状态存在细微差异。
- 性能风险：极端长文页面在低性能设备上可能接近 1 秒阈值。

## 4. 验收口径（DoD）

- 功能完成：支持多意图模式、站点默认意图、DOM 重排与一键还原。
- 测试通过：单元/集成/性能测试覆盖并全部通过。
- 性能达标：页面重写链路（不含 AI）在基准测试内小于 1 秒。
- 文档更新：README、architecture、assumptions、changelog 与阶段计划同步更新。
- 可回滚明确：重写过程记录可逆变更并可在路由切换或手动触发时可靠恢复。
