# 集成需求文档：知识织网者 + 网页变形金刚 + AI浏览器伴侣

## 1. 项目概述

### 1.1 产品名称
**NexusMind**（暂定）

### 1.2 产品愿景
为用户打造一个**智能化的浏览器工作台**，不仅能自动提取、关联跨网页的知识，还能根据用户意图重构页面阅读体验，并支持自然语言对话式深度理解。让浏览器从“信息容器”进化为“思维伙伴”。

### 1.3 目标用户
- 研究人员、学生：需要从大量文献、技术文档中提炼知识脉络。
- 知识工作者：经常在多网页间切换、整理信息（如产品经理、分析师）。
- 终身学习者：希望通过个性化阅读加速理解。
- 开发者：阅读技术教程、代码库，需要快速提取核心。

### 1.4 核心价值
- **知识管理**：自动构建个人知识图谱，连接不同页面中的概念、观点、矛盾。
- **自适应阅读**：根据用户当前目的（新手/专家/只看代码/只读结论）重塑页面内容。
- **智能对话**：针对当前页面或整个知识库进行问答，并联动高亮原文。

## 3. 功能需求

### 3.1 模块一：知识织网者（跨网页知识图谱）

#### 3.1.1 知识提取
- **自动实体识别**：当用户浏览网页时，插件自动识别页面中的关键实体（人物、技术、产品、概念、组织等），并提取实体间的显式/隐式关系（如“A 依赖于 B”、“X 反驳 Y”）。
- **用户可干预**：允许用户手动标记重要段落或关系，或对自动提取的结果进行编辑（增加、删除、修改实体/关系）。
- **多语言支持**：支持中英文及主流语言实体识别。

#### 3.1.2 知识存储与图谱构建
- **本地知识库**：所有知识存储在用户本地的 IndexedDB 中，不上传云端。
- **图谱结构**：实体作为节点，关系作为边，支持属性（如“出现次数”、“最后浏览时间”）。
- **跨网页关联**：同一实体在不同页面出现时，自动合并为一个节点，并记录来源页面链接。

#### 3.1.3 可视化与浏览
- **侧边栏图谱**：在浏览器侧边栏中展示知识图谱的缩略图，支持缩放、拖拽。
- **节点详情**：点击节点可查看所有相关的页面列表、提取的关系、原文摘要。
- **路径探索**：支持查看两个实体之间的连接路径（如“A -> B -> C”）。

#### 3.1.4 搜索与回溯
- **全局搜索**：在侧边栏搜索框输入实体名或关键词，快速定位到相关节点和页面。
- **时间线回溯**：按时间顺序展示用户对某实体的浏览历史。

---

### 3.2 模块二：网页变形金刚（意图驱动页面重写）

#### 3.2.1 用户意图选择
- **意图面板**：在侧边栏顶部提供意图切换按钮，包括：
  - **新手模式**：保留基础概念、添加术语解释、简化语言。
  - **专家模式**：隐藏基础知识，突出高级观点、细节讨论。
  - **只看结论**：仅显示文章的总结、关键数据、最终结论。
  - **只看代码**（针对技术页面）：提取所有代码块，删除正文说明。
  - **只看数据/表格**：突出显示图表、数据表格、统计数据。
- **自定义意图**：用户可通过自然语言描述自己的阅读目的（如“我想知道这个产品的缺点”），AI 自动生成对应的页面重写策略。

#### 3.2.2 页面内容重组
- **动态 DOM 重排**：根据意图，隐藏或重新排列页面元素（段落、代码块、图表），保留核心内容。
- **添加辅助信息**：在新手模式下，为专业术语添加工具提示（悬浮显示定义）；在专家模式下，可添加引用链接或深度讨论。
- **布局切换**：支持将长文章转换为问答形式、卡片式摘要等。

#### 3.2.3 意图记忆
- **站点级配置**：用户可以为特定域名（如 `medium.com`、`github.com`）设置默认意图，下次访问时自动应用。
- **临时覆盖**：用户可随时手动切换意图，不影响默认设置。

#### 3.2.4 恢复原页面
- **一键还原**：在侧边栏提供“还原页面”按钮，恢复原始 DOM 结构。

---

### 3.3 模块三：AI浏览器伴侣（上下文问答与联动）

#### 3.3.1 基于当前页面的问答
- **侧边栏聊天界面**：用户可输入自然语言问题，AI 基于当前页面的内容（结合提取的文本、图片、代码）生成回答。
- **答案联动**：回答中提及的原文片段，点击后自动滚动到页面相应位置并高亮。
- **多轮对话**：支持上下文对话，记忆之前的问题与答案。

#### 3.3.2 基于知识图谱的问答
- **跨页面问答**：用户可询问涉及之前浏览过的内容，AI 检索知识图谱中的相关实体和关系，结合多个页面内容回答。
- **示例**：“我之前看到的关于‘量子计算’的几种技术路线有哪些矛盾点？” —— AI 会从图谱中找到相关实体和关系，综合回答。

#### 3.3.3 智能推荐与提醒
- **主动提醒**：当用户正在浏览的内容与知识图谱中已有的实体高度相关时，插件可提示“您之前看到过关于这个主题的 X 个页面，是否回顾？”
- **建议探索**：基于当前实体，推荐图谱中与之关联但尚未浏览的实体。

#### 3.3.4 对话历史与导出
- **保存对话**：用户可保存问答历史，便于回顾。
- **导出**：支持将对话记录、知识图谱导出为 JSON、Markdown 或图片。

---

### 3.4 共享与整合功能

#### 3.4.1 统一设置
- **全局设置面板**：用户可以统一配置 API Key（支持 OpenAI、Claude、Gemini 等）、模型选择、隐私选项。
- **成本控制**：设置每日/每月 API 调用额度，达到阈值后提示或暂停高级功能。

#### 3.4.2 多模态支持
- 对于包含图片的页面，支持将图片作为上下文传递给 AI（需用户授权）。
- 在问答中，AI 可描述图片内容；在知识图谱中，图片可作为节点属性存储。

#### 3.4.3 快捷键与交互
- 提供全局快捷键（如 `Ctrl+Shift+K` 打开侧边栏）快速调用各功能。
- 右键菜单集成常用操作（如“将选中文字加入知识图谱”、“以此意图重写页面”）。

## 4. 非功能需求

### 4.1 性能
- 页面重写应在 1 秒内完成（不包含 AI 调用时间）。
- 知识图谱可视化支持至少 2000 个节点时保持流畅交互。
- AI 调用采用流式响应，逐步显示结果，避免长时间等待。

### 4.2 隐私与安全
- 所有用户数据（知识图谱、页面内容、对话记录）**仅存储在用户本地**，不上传任何第三方服务器。
- API Key 加密存储（`chrome.storage.local`）。
- 页面内容仅在用户主动触发时发送给 AI 服务商，并提示可能产生的费用。
- 用户可选择完全离线模式（使用本地小模型进行实体识别和问答，但功能受限）。

### 4.3 兼容性
- 支持 Chrome 和 Edge（基于 Chromium）浏览器。
- 兼容主流网站（包括单页应用，通过监听 DOM 变化）。
- 支持 Manifest V3。

### 4.4 可扩展性
- 插件架构允许后续增加新的意图模板、图谱分析算法、问答模型等。
- 提供开发者 API 供第三方扩展（可选）。

## 5. 用户界面设计（概要）

- **侧边栏**：主界面，包含四个选项卡：
  1. **图谱**：知识图谱可视化区域，附带搜索框。
  2. **意图**：意图切换按钮组，以及当前页面重写预览（可选）。
  3. **对话**：聊天界面，显示问答历史。
  4. **设置**：全局配置、模型选择、费用管理。
- **悬浮按钮**：在页面右下角或右上角提供快捷入口，一键打开侧边栏或执行常用意图。
- **右键菜单**：集成“加入知识图谱”、“使用此意图重写”、“询问 AI”等选项。
- **页面内高亮**：问答联动时高亮原文片段；知识图谱节点关联时，在页面中标记相关段落。

## 6. 技术架构（简要）

### 6.1 前端
- **Manifest V3**：background service worker, content script, side panel, popup。
- **框架**：可使用 Vue/React 构建侧边栏 UI。
- **知识图谱**：`d3.js` 或 `vis-network` 进行可视化。
- **数据存储**：IndexedDB（使用 Dexie.js 封装）存储实体、关系、页面元数据。

### 6.2 AI 能力
- **实体识别**：可使用本地 `transformers.js` 加载轻量 NER 模型（如 `bert-base-NER`）实现离线实体识别，或通过 API 调用更强大的模型（用户自选）。
- **关系提取**：通过 API 调用（用户 Key）实现，prompt 设计。
- **页面重写**：调用 API 生成重写策略（可复用同一个模型），由 content script 执行 DOM 操作。
- **问答**：通过 API 流式调用，同时检索知识图谱或当前页面文本作为上下文。

### 6.3 通信
- background 作为代理，统一管理 API 请求，存储敏感信息。
- content script 负责 DOM 操作、页面内容提取、用户交互。
- side panel 通过消息传递与 background、content script 通信。

## 7. 数据模型（主要实体）

### 7.1 实体（Entity）
- `id`：唯一标识
- `name`：实体名称
- `type`：类型（人、技术、产品、概念等）
- `aliases`：别名列表
- `createdAt`：首次发现时间
- `updatedAt`：最后更新时间

### 7.2 关系（Relation）
- `id`
- `sourceId`：源实体 ID
- `targetId`：目标实体 ID
- `type`：关系类型（如“依赖”、“反驳”、“支持”、“定义”等）
- `description`：关系描述（来源于原文摘要）
- `sourceUrl`：来源页面 URL
- `createdAt`

### 7.3 页面（Page）
- `url`：页面 URL
- `title`：标题
- `extractedAt`：提取时间
- `contentHash`：内容哈希，用于判断页面是否更新
- `entities`：包含的实体 ID 列表

### 7.4 意图配置（IntentConfig）
- `urlPattern`：匹配的 URL 正则或域名
- `defaultIntent`：默认意图（新手、专家等）
- `customPrompt`：可选自定义意图 prompt

## 9. 风险与应对

| 风险                 | 应对措施                                                     |
| -------------------- | ------------------------------------------------------------ |
| API 调用成本过高     | 提供成本控制设置、本地模型降级、用户明确授权                 |
| 隐私担忧             | 所有数据本地存储，不开源关键部分，提供隐私说明               |
| 网站兼容性问题       | 逐步增加 fallback 机制，支持单页应用，提供用户反馈入口       |
| 知识图谱复杂度过高   | 初期限制节点数量，提供清理过期数据功能                       |
| 意图重写导致页面失效 | 提供一键还原，限制重写范围（只隐藏/显示元素，不删除关键交互） |

## 10. 附录

- **实体**：现实世界中的对象或概念，如人物、技术、产品。
